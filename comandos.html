Curso Linux

Quando logamos no sistema Linux, nós caímos no diretório do usuário usado para logar.

Exemplo: Se logarmos como root caímos em um diretório vazio, que não tem nada.

Para ver os arquivos principais do sistema usamos:

ls / ( / = (barra) diretório do sistema – Diretório raiz)

* Alguns comandos vêm com o sistema e outros não.

Variáveis na linha de comando Shell

Exemplo: * Coloca-se cifrão para acessar a variável

~# cor=azul (sem espaços)
 ~# echo $cor
 ~# azul (resultado)

* Para que o Linux encontre os comandos, estes precisam estar listados na variável PATH Exemplo:

echo $PATH Resultado: /usr/local/sbin:/usr/local/sbin:………

* Existem comandos que só aparecem para o root
Ex: halt /sbin só é executado pelo root

Para localizar comandos ou seja saber onde os comandos estão:

Ex: which ls Resultado: /usr/bin/ls

Ex: which shutdown Resultado: /usr/sbin/shutdown

* shutdown Só aparece se estiver logado como root
* Usuário comum não desliga o sistema





Quanto trabalhamos com comandos nós temos as opções dos comandos:

Ex: cp -rfv /etc/root
cp = comando
-rfv = Opções
/etc/root = Parâmetros

* Quando criar arquivos no Linux não use espaços, pois, espaços fazem parte de comandos no Shell.

* Podemos usar caracteres especiais que podem substituir caracteres.

Ex: ls /etc/host* (vai procurar dentro do diretório /etc tudo que tiver host)

* ? (Interrogação) também é um caractere especial) Ex: ls /dev/sda?


A interrogação substitui apenas um caractere, neste caso vai procurar tudo que for SDA? alguns coisa Ex: /dev/sda1 - /dev/sda2 - /dev/sda3

* Também é possível usar colchetes []

ls /dev/sd[abc] resultado /dev/sda (procura tudo que for sd alguma coisa

* Também é possível usar chaves{}

Ex: ls /dev/{sda,null} Resultado: /dev/null /dev/sda

Outras formas de usar ls

ls -l /etc/ - mostra em lista
ls -a /etc/ - mostra arquivos ocultos

* Tudo que tiver . (ponto) na frente se torna um arquivo oculto. Ex. .profile

ls –-color /etc/ (Mostra de forma colorida), assim diferencia arquivo, diretório e link.

ls –-color -F /etc (O F adiciona uma informação na frente da barra. Ex:
/ diretório
@ link
password
rc.local*



ls -laF –-color /etc
l = lista
a = Oculto
F = Identificando a característica de cada arquivo

Digitar comandos em sequência

; (ponto e virgula) executa os comandos independente se o último foi executado com sucesso ou não.

Ex: cls; cd /etc; pwd (Resultado: comando não encontrado, mas entra na pasta /etc)


& e comercial

cls && cd /etc && pwd (Executa o comando seguinte apenas se o anterior foi bem-sucedido)

|| Duplo tipe (Executa o comando seguinte apenas se o anterior não foi executado com sucesso.

cls || cd /etc || pwd

* TAB – Completa o nome dos arquivos, se apertar mais vezes, mais mostrando mais opções

* HELP – mostra todos os comandos (Exemplo: comando + help)

* man + comando (Exemplo: man ls )





















CHMOD versus CHOWN

Permissões de arquivos


O comando chmod permite alterar as permissões de arquivo, enquanto chown altera o dono do arquivo.

Inserindo todas as permissões a um arquivo


chmod 777 arquivo.txt



O que significa cada número?













Mudando o dono de um arquivo


chown diego:diego arquivo.txt



O parâmetro -R faz com que a alteração seja recursiva, isto é, afete todas as pastas e arquivos dentro de uma pasta especificada.

chmod -R 777 *

Neste exemplo o * indica tudo presente no diretório.
